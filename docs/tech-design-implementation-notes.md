# 元素尖塔 2 技术文档对照开发记录

## 本轮已落地

- 对齐技术文档 `3.6 遗物 / 圣遗物系统`，新增第一版遗物系统。
- 新增遗物数据池，覆盖战斗开始、回合开始、元素反应、战斗胜利等触发点。
- 战斗胜利奖励从“金币 + 三选一卡牌”扩展为“金币 + 三选一卡牌 + 1 件遗物”。
- 侧栏新增当前遗物列表，奖励界面展示本次获得的遗物。
- 自检面板新增遗物奖励与遗物触发测试。
- 对齐技术文档 `5.2 效果解释器`，新增第一版卡牌效果解释器。
- 现有旧字段卡牌会自动转换为效果列表，未来新卡可以直接使用 `effects` 数组。
- 对齐技术文档 `6.4 IntentSystem` 和 `3.5 敌人系统`，新增第一版敌人意图系统。
- 敌人现在支持攻击、防御、攻防混合三类意图，敌人护盾会抵消玩家伤害。
- 对齐技术文档 `3.7 地图与事件系统`，新增事件、商店、休息点、宝箱节点。
- 路线选择现在不只进入战斗，也会进入非战斗节点并在结算后继续前进。
- 对齐技术文档 `3.3 角色系统`，新增 3 个单角色原型与角色选择面板。
- `createRun(characterId)` 会按角色生命、费用、初始卡组和被动创建本局。

## 建议写回技术文档的增补

### 3.6 遗物 / 圣遗物系统

建议补充第一版触发器范围：

- `onCombatStart`：战斗开始时触发。
- `onTurnStart`：玩家回合开始时触发。
- `onReaction`：元素反应发生时触发。
- `onVictory`：战斗胜利时触发。

建议补充第一版遗物效果范围：

- 获得护盾。
- 获得费用。
- 抽牌。
- 获得金币。
- 恢复生命。

### 10. MVP 范围

原文写 “20 个遗物”。当前 Demo 建议分两步：

- Demo 骨架阶段：先实现 6 个遗物，验证触发链路。
- MVP Demo 阶段：扩展到 10 个遗物。
- 垂直切片阶段：扩展到 20-50 个遗物，并加入稀有度权重。

### 13. 推荐开发顺序

当前项目已经完成：

- 抽牌、打牌、弃牌、费用。
- 基础敌人行动。
- 基础元素附着与反应。
- 地图节点雏形。
- 奖励系统雏形。
- 小地图常驻进度。
- 第一版遗物触发。
- 第一版卡牌效果解释器。
- 第一版敌人意图系统。
- 第一版事件、商店、休息、宝箱节点。
- 第一版角色选择、角色初始卡组、角色被动。

建议下一步顺序：

1. 加入本地存档，保存当前 run 与解锁进度。
2. 将单敌人战斗扩展为 `enemies[]`，为超载、扩散、感电等群体效果做准备。
3. 逐步把卡牌数据迁到显式 `effects` 数组，减少旧字段依赖。
4. 在意图系统中补充强化、施加状态、蓄力和元素爆发。

## 暂不建议改动的内容

- 暂不引入三人小队制，当前 Demo 仍应保持单角色制。
- 暂不做完整草系反应链，链式状态会放大战斗解释器复杂度。
- 暂不把项目迁移 TypeScript，先保持 React/Vite 原型快速迭代。
